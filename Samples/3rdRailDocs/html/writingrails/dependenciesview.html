The Dependencies View

Rails makes doing common tasks relatively easy but adds a level of complexity by adding relationships and dependencies not obvious through use of conventions, meta programming and naming patterns. The Dependencies View keeps a live view of class and logical relationships. While navigating source code or nodes in the Rails Explorer, use the Dependencies View to track focus and show the dependencies (i.e., class, method, controller, action, view, attributes). The Dependency View reveals references and from a selected class or method. This is a very useful tool for navigating and introspecting software. Relationships are determined by type inference and semantic analysis. Dependencies are updated in real-time during editing and navigation. Relationships, between controllers, actions, views, models, helpers, routes, migrations, schema and tests are related and determined by convention and method call analysis.
